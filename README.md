# Resume YAML Generator

A Python tool to generate professional PDF resumes from YAML files using RenderCV.

## Project Structure

```
resume_yaml/
├── main.py                 # Main script to generate resumes
├── read_templates.py       # Template selection utility
├── requirements.txt        # Python dependencies
├── designs/                # Design template YAML files (templates)
│   └── clean_look.yaml     # Default design template
├── accadamic/              # Jinja templates used by RenderCV
├── resume_yaml/            # Example resume YAML files (user-provided)
├── output/                 # Generated intermediate files (auto-created)
└── resumes/                # Final PDF output (auto-created)
```

## Prerequisites

- Python 3.7 or higher
- pip (Python package manager)

## Installation

1. **Navigate to the project directory:**


## Usage

### Basic Command

```bash
python3 main.py <input_yaml_file> <output_filename> [design_index]
```

### Parameters

- `<input_yaml_file>` - Path to your resume data YAML file (e.g., `Ramana.yaml`)
- `<output_filename>` - Desired output PDF filename (e.g., `Ramana_Gangarao_Resume.pdf`)
- `[design_index]` - Optional design template index (defaults to 1 if not specified)

### Examples

**Generate resume with default design (design #1):**
```bash
python3 main.py resume_yaml/Ramana.yaml Ramana_Gangarao_Resume.pdf
```

**Generate resume with specific design template:**
```bash
python3 main.py Ramana.yaml Ramana_Gangarao_Resume.pdf 1
```

**List available design templates:**
The script will automatically list available designs when run:
```
YAML files in ./designs:
1. clean_look.yaml
```

**Generate resume for another person:**
```bash
python3 main.py resume_yaml/Ramana.yaml Ramana_Resume.pdf
```

## How It Works

1. **Load Resume Data**: Reads your resume information from the input YAML file
2. **Load Design Template**: Selects and loads the design template from `designs/` folder
3. **Merge Configuration**: Combines resume data with design settings
4. **Generate PDF**: Uses RenderCV to create the PDF resume
5. **Save Output**: Moves the final PDF to `resumes/<output_filename>/` directory

## Customizing Your Resume

### Edit Resume Content

Edit your resume YAML file (e.g., `Ramana.yaml`) with your information:

```yaml
cv:
  name: Your Name
  location: Your Location
  email: your.email@example.com
  phone: +1 (123) 456-7890
  sections:
    experience:
      - company: Company Name
        position: Your Position
        start_date: 2024-01
        end_date: present
        highlights:
          - Achievement 1
          - Achievement 2
```

### Customize Design

Edit `designs/clean_look.yaml` to change:
- Theme
- Margins
- Spacing between entries
- Header icons
- Colors and fonts

## Output Locations

- **Generated PDF**: `./resumes/<output_filename>/<output_filename>.pdf`
- **Intermediate files**: `./output/file_<design>_<timestamp>/` (auto-generated)

## Cleanup before pushing

Before you push the repository to a remote, stop tracking autogenerated files and caches. Recommended steps:

```bash
# Remove generated files from the index (they will remain locally and be ignored):
git rm -r --cached output resumes || true

# Remove compiled caches from the working tree (these will be ignored after .gitignore):
find . -name "__pycache__" -type d -exec rm -rf {} + || true
find . -name "*.pyc" -delete || true

# Commit the cleanup and push
git add .gitignore README.md
git commit -m "chore: add .gitignore and remove generated files"
git push origin main
```

Avoid committing personal backups, zip files, or large PDFs. If large generated files were already committed, use `git filter-repo` (preferred) or `git filter-branch` to purge them from history.

## Troubleshooting

**Error: "ModuleNotFoundError: No module named 'yaml'"**
```bash
pip3 install pyyaml
```

**Error: "rendercv: command not found"**
```bash
pip3 install rendercv[Full]
```

**Error: Design template not found**
- Ensure `designs/clean_look.yaml` exists
- Check the design_index parameter (should be 1 for first design)

## Design Template Configuration

The `clean_look.yaml` design file supports:
- `theme`: RenderCV theme (e.g., `sb2nov`)
- `disable_last_updated_date`: Hide date in header
- `header.connections.show_icons`: Show icons for phone/email/location
- `margins`: Page margins and spacing settings
- `locale.language`: Language for date formatting
